### 난이도

### 알고리즘 분류

- 정렬
- 이분 탐색
- 투 포인터

### 핵심 아이디어

배열 돌면서 다른 수들의 합이 해당 요소가 될 수있는 조합 찾기

-> 투포인터로 각각 왼, 오로 돌면서 가능한 조합 판별 => O(N^2)

### 로직

    while(l < r)
    cur = a[i]
    l, r != cur
    sum = a[l] + a[r]

    만약 sum > cur
        r--
    sum < cur
        l++

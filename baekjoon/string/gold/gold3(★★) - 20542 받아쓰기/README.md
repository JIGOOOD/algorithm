### 난이도

### 알고리즘 분류

- 다이나믹 프로그래밍
- 문자열

### 핵심 아이디어: 편집거리 알고리즘을 사용하자

- 편집거리 알고리즘: 두 문자열을 같게 만들기 위해 필요한 최소 수정 횟수를 구하는 알고리즘

- dp[i][j] = word1(답안)의 앞 i글자를 word2(정답)의 앞 j글자로 만드는 최소 편집 횟수

- 표를 채울 때 dp[i][j]는 주변의 세 칸(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) 중 가장 작은 값에 기반해 결정됨

- 방향별 설명

  ① 왼쪽에서 오는 경우 (dp[i][j-1] + 1) : 문자 추가답안은 그대로인데 정답의 글자가 하나 늘어난 상황. 즉, 부족한 글자를 하나 '추가'해서 맞췄다는 뜻

  ② 위쪽에서 오는 경우 (dp[i-1][j] + 1) : 문자 삭제정답은 그대로인데 답안의 글자만 하나 늘어난 상황.즉, 쓸데없는 글자를 하나 '삭제'해서 맞췄다는 뜻

  ③ 대각선 위에서 오는 경우 (dp[i-1][j-1] 또는 +1) : 문자 수정/유지답안과 정답 모두 글자가 하나씩 늘어난 상황.
  - 글자가 서로 다를 때 (+1): 이 글자를 정답 글자로 '변환(수정)'했다는 뜻
  - 글자가 서로 같을 때 (+0): 수정할 필요가 없으므로 이전까지의 비용을 그대로 가져옴

- 휘갈겨 쓴 애면 대각선 그대로 가져오고, 휘갈겨 쓴 애가 아니라면 값이 같으면 대각선 그대로, 다르면 인접한 세 값 중 최솟 값 + 1

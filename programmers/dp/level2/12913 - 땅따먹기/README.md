## **💻 문제 정의**

땅따먹기 게임을 하려고 합니다. 땅따먹기 게임의 땅(land)은 총 N행 4열로 이루어져 있고, 모든 칸에는 점수가 쓰여 있습니다. 1행부터 땅을 밟으며 한 행씩 내려올 때, 각 행의 4칸 중 한 칸만 밟으면서 내려와야 합니다. **단, 땅따먹기 게임에는 한 행씩 내려올 때, 같은 열을 연속해서 밟을 수 없는 특수 규칙이 있습니다.**

예를 들면,

| 1 | 2 | 3 | 5 |

| 5 | 6 | 7 | 8 |

| 4 | 3 | 2 | 1 |

로 땅이 주어졌다면, 1행에서 네번째 칸 (5)를 밟았으면, 2행의 네번째 칸 (8)은 밟을 수 없습니다.

마지막 행까지 모두 내려왔을 때, 얻을 수 있는 점수의 최대값을 return하는 solution 함수를 완성해 주세요. 위 예의 경우, 1행의 네번째 칸 (5), 2행의 세번째 칸 (7), 3행의 첫번째 칸 (4) 땅을 밟아 16점이 최고점이 되므로 16을 return 하면 됩니다.

#### 제한사항

- 행의 개수 N : 100,000 이하의 자연수
- 열의 개수는 4개이고, 땅(land)은 2차원 배열로 주어집니다.
- 점수 : 100 이하의 자연수

#### 입출력 예

| land                            | answer |
| ------------------------------- | ------ |
| [[1,2,3,5],[5,6,7,8],[4,3,2,1]] | 16     |

#### 입출력 예 설명

입출력 예 #1

문제의 예시와 같습니다.

## **💡 접근 및 설계**

모든 가능한 수를 구하자. 배열 값이 10만 \* 4이므로, 상하로 인접한 배열을 n번 돌며 모든 값을 구하는 시간 복잡도가 16N이므로 풀릴 것이다!

### **✏️ 알고리즘 풀이**

따로 dp 배열 생성 후, 각 행이 가질 수 있는 최댓값 구하기. 이를 n번 적용.

dp[i][j] = 이곳에 도달할 수 있는 위에서부터 누적된 최댓값

### **🗒️ 최종 제출 코드**

```python
def solution(land):
    n = len(land)
    dp = [[0] * 4 for _ in range(n)]
    dp[0] = land[0][:]

    for i in range(1, n):
        for j in range(4):
            for k in range(4):
                if j == k: continue
                dp[i][k] = max(dp[i][k], dp[i-1][j] + land[i][k])

    return max(dp[n-1])
```

웬걸! 정확도 테스트는 다 맞았는데, 효율성 테스트에서 테케 4개 중 2개가 시간 초과가 났다. 그럴리가 없는데 말이다! 160만번 연산인데.. 절대 그럴리가 없는데.. 왜 시간초과가 나지? 말이 안되는데.. 멘붕이 왔다!

그래서 왜지? 이게 파이썬의 단점인가! 하면서 곰곰이 생각해봤다. 근데 도무지 다른 방안이 안떠오르는 것이다! 그래서 억울해서 C++로 똑같이 제출해봤는데 통과됐다.

파이썬은 어떻게 풀어야 하는지 궁금해서 인터넷에 검색해서 답을 봤다. 그리고 내식대로 바꿔서 다시 제출했다.

```python
def solution(land):
    n = len(land)
    for i in range(1, n):
        for j in range(4):
            land[i][j] += max(land[i-1][:j] + land[i-1][j+1:])

    return max(land[n-1])
```

신박한 방법이었다.

그런데 어차피 max 배열의 값은 3개가 있고 어차피 한번에 12번 연산하고, 내 코드도 위아래 인덱스 겹치는거 빼면 12번인데 무슨 차이인지 모르겠다.

그래서 알아보니, 정답 코드는 반복문을 한번만 돌고, 최댓값을 한번에 돌아서 연산이 적은 반면, 내 코드는 2중 반복문에 max 값을 계속 계산하니 연산비용이 많이 든다고 한다. 제미나이가 말해준거라 맞는지는 모르겠다.
